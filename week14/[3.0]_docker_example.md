# Docker Deployment Guide (Week 14)

## Overview
This guide explains how to package the Observable Agent API into a Docker container for production deployment.

## 1. Create Dockerfile
Create a file named `Dockerfile` in the root directory:

```dockerfile
# Use minimal Python image
FROM python:3.11-slim

# Set working directory
WORKDIR /app

# Copy requirements FIRST (for caching)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Copy application code
COPY week8/ week8/
COPY week13/ week13/
COPY week14/ week14/
COPY .env .env

# Expose API Port
EXPOSE 8000

# Run API with Uvicorn
CMD ["uvicorn", "week14.[1.0]_agent_api:app", "--host", "0.0.0.0", "--port", "8000"]
```

## 2. Build Image
Run the following command in the terminal:
```bash
docker build -t observable-agent:v1 .
```

## 3. Run Container
Start the container and map port 8000:
```bash
docker run -d -p 8000:8000 --name agent-api observable-agent:v1
```

## 4. Test
Verify the service is running:
```bash
curl http://localhost:8000/health
```

## 5. Security Note - Be Careful!
- **Never commit .env files** to Git. Use environment variables in Docker Compose or Kubernetes Secrets instead.
- Pydantic Settings is recommended for managing secrets securely.
